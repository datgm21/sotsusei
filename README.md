## 10/21の試遊＆プレゼン会について
- 10/20(木)に予定されていた試遊会と11/4(金)のプレゼンが分かれていると作業時間が取られて勿体ないという理由で、10/21(金)にまとめて実施することに変更
- 10/21の試遊会がDATフェスタ前に他チームのメンバーに見てもらえる最後の機会である。これを逃すとゲームを知らない人が企画の意図に気付けるかや、操作性に問題がないか、伝わりにくいところがどこかなどを知ることができなくなる。この段階でゲームとして必要な要素は全て盛り込んでおく必要がある
- これ以降は原則として、バランス調整、テストとデバッグ、クオリティアップの期間になる。残りの期間から実装するものと見送るものを見極めて、最小限の展示可能バージョンをまとめるための計画を考えること

- 目的
  - 開発メンバー以外の人達に、企画の狙いや遊び方が伝わっているかを確認する
- 必要なもの
  - ビルドファイル一式
  - 簡単なゲーム概要と操作説明を書いたテキストファイル
    - ビルドフォルダーに`README.txt`というテキストファイルを作成して、そこに記載
    - unityroomの説明参考
  - ESCキーで終了するようにする
- データの提出先
  - `X:\2022年\ゲーム学科2\平井・田中・細谷\1st_試遊会`フォルダー内に作品名のフォルダーを作成して、そこにビルドしたファイル一式とREADME.txtをコピー
- 明日
  - ※ネットドライブから直に起動すると遅いので、試遊時は各自のPCのデスクトップにデータをコピーしてから、コピーしたファイルを起動する
  - 試遊中に、メモ帳などのテキストファイルで、遊んだ感想や気づいた不具合、改善案などを記入する
    - 提出先 `X:\2022年\ゲーム学科2\平井・田中・細谷\1st_試遊会\みんなの感想`の中のグループ名のフォルダー内
    - テキストファイル名は、出席番号と氏名にする(例 `00tanaka.txt` など)

- 試遊できるものをビルドして、共有ドライブにコピー
- コピーしたフォルダー内に`README.txt`ファイルを用意
